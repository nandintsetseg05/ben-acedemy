You are a senior C# full-stack developer. 
I want you to create **Sprint 3 of the Ben Academy MVP** using ASP.NET Core Web API with Blazor WebAssembly frontend. The focus is **Teacher Marketplace, 1:1 bookings, and notifications**. Follow these steps carefully:

---

1. **Teacher Profiles**
   - EF Core `Teacher` model:
     - Id (GUID)
     - UserId (FK)
     - IELTSScore (float)
     - Bio (string)
     - HourlyRate (decimal)
     - AvailableTimes (JSON or structured schedule)
   - Backend API endpoints:
     - `GET /api/teachers` → list teachers (filter by IELTS score, availability)
     - `GET /api/teachers/{id}` → teacher details
     - `PUT /api/teachers/{id}` → teacher updates profile
   - Frontend:
     - Blazor page to browse teachers
     - Filter/search teachers
     - View profile with bio, IELTS score, rate, availability

---

2. **1:1 Booking System**
   - EF Core `Booking` model:
     - Id (GUID)
     - StudentId (FK)
     - TeacherId (FK)
     - Time (DateTime)
     - Status (`Pending`, `Confirmed`, `Declined`, `Completed`)
     - PaymentStatus (`Free`, `Paid`)
   - Backend endpoints:
     - `POST /api/bookings` → student requests session
     - `PUT /api/bookings/{id}` → teacher accepts/declines
     - `GET /api/bookings/student/{id}` → student sees requests
     - `GET /api/bookings/teacher/{id}` → teacher sees requests
   - Frontend:
     - Booking page with calendar/time picker
     - Teacher accepts/declines requests
     - Free first 3 sessions logic implemented automatically

---

3. **Notifications**
   - Notify students when:
     - Booking confirmed
     - Teacher leaves comment
     - Session approaching (optional)
   - Notify teachers when:
     - Student requests session
   - Simple notification system:
     - EF Core `Notification` model: Id, UserId, Message, IsRead, CreatedAt
     - Frontend notification panel in dashboard

---

4. **Security & Access Control**
   - Role-based API access:
     - Only **students** can request bookings
     - Only **teachers** can accept/decline
   - Validate schedule/time conflicts
   - JWT authentication for all endpoints
   - Input validation using FluentValidation / DataAnnotations

---

5. **Stripe Integration for Paid Sessions**
   - Paid sessions processed via Stripe.NET

     - Add availability

7. **Project Structure**
